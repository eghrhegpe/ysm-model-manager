# YSM 模型管理器

类似 Mod Organizer 2 的 Minecraft YSM 模型管理器。
灵感来自：LytVPK mod-manager

**技术栈：** Go (Wails v2) + 原生 HTML/CSS/JS

---

## ✅ 已完成功能

### 仓库管理

- 扫描模型文件（.ysm / .zip / .7z）
- 仓库树浏览（文件夹 / 文件）
- 搜索 + 排序（名称 / 大小 / 日期）【修复：Go 端移除硬编码名称排序，前端接管排序逻辑，按选择实时重排】
- 启用 / 禁用（`.ban` 后缀切换，复选框）
- 去重（按 SHA256 哈希，保留一个其余移入 `.recycle`）
- 右键菜单：复制文件名、复制哈希、打开文件夹、去重信息

### 整合包管理

- 自动列出版本实例（`versions/` 目录）
- 检测 YSM 模组（支持 `mods.toml` + `neoforge.mods.toml`）
- 同步状态显示：✅ 已同步 / ⬇️ 缺失 / ⚠️ 禁用 / 📤 额外
- 整合包卡片展开/折叠 + `localStorage` 持久化
- 整合包搜索 + YSM 模组筛选
- 排序：有 YSM 优先 → 缺失数升序

### 同步与安装

- 批量安装（仅缺失）：自动识别缺失模型安装到对应整合包
- 全覆盖安装：仓库所有模型 → 所有整合包
- 上传新模型到仓库（整合包 Extra → 仓库）
- 进度显示（当前整合包 / 总文件 / 成功数）
- 安装失败 toast 提示 + 详细清单弹窗
- 链接模式：复制 / 硬链接 / 符号链接（默认复制）

### 回收站

- 去重 / 卸载文件自动移入 `.recycle`
- 符号链接：直接删除（不进回收站）
- 硬链接：直接删除（数据还在仓库）
- 跨分区文件：直接删除（无法移入回收站时）
- 路径遍历防护（`ensureInDir` 检查）
- 恢复 / 永久删除 / 清空

### 一键上传非仓库模型

- 统计面板 `📤 上传待上传` 按钮，一键扫描所有整合包 Extra 中的非仓库模型
- 自动跳过同名/同哈希，完成后自动刷新仓库树 + 同步状态
- 显示上传成功/失败统计清单

### 导入

- 拖拽导入（Base64 传输 + 文件校验：魔数 / 大小 / 扩展名）
- 导入日志（成功 / 失败 / 跳过）

### UI

- 三栏布局（侧栏 / 仓库树 / 统计面板），支持侧栏和预览面板折叠展开
- 明暗主题切换（按钮颜色随主题变化）
- 右键菜单
- 窗口位置记忆（`window_state.json`）
- 统计面板：仓库模型数 / 整合包数 / 完全同步 / 待上传模型
- 防缓存刷新（`<script src="...?v=...">`）
- 仓库搜索高亮：搜索词匹配部分用 `<mark>` 包裹，有搜索词时文件夹自动展开
- 文件夹展开/折叠状态持久化到 `localStorage`

### 配置持久化

- 磁盘 JSON 文件（`ysm_config.json`）：保存 repoRoot、mcRoot、linkMode
- 启动时从磁盘读取，失败回退 `localStorage`
- 目录选择 / 链接模式切换时自动保存
