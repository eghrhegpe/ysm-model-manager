# v1.0.6 — 旧帝国清理 + 拖拽导入封板

> 发布日期: 2026-06-07

## 🎯 核心改动

### 📥 全局拖拽导入（封板）

- 支持单文件、多文件、文件夹拖入（递归遍历子目录）
- 遮罩正确显示"📥 放开以导入模型"，放下后立即消失
- WebView2 安全策略兼容：`dragover` 阶段不读文件信息，`drop` 阶段用 `items` + `webkitGetAsEntry()` + `files` 三重回退
- `DataTransferItem` 无 `.name` 属性的兼容处理
- 回调式 `FileSystemEntry.file(callback)` 全部 Promise 化
- 文件大小熔断（>10MB）+ 数量熔断（>50 个）

### 📂 右键菜单"移动到文件夹"

- 文件右键新增"移动到…"，弹窗输入目标文件夹名即可移动
- 批量多选支持"移动到…"
- 调用 Go 端 `MoveModelFile(src, dstDir)`

### 🧹 旧帝国清理

- 删除 `app-legacy-bundle.js`（2198 行旧版全局代码）及 `index.html` 引用
- 删除 `creator-manager.js`（412 行死代码，从未被 import）
- 删除 `public/js/app-legacy-bundle.js`（Vite dev 模式幽灵副本）

### 📦 代码文件按职责拆分

- `global-handlers.js`（579行）→ `handler-dnd.js` + `handler-sync.js` + `handler-upload.js` + `handler-other.js`
- `app-modules.js`（339行）→ 右键菜单提取到 `core/context-menus.js`（140行）

## ⚠️ 破坏性变更

- 移除 `app-legacy-bundle.js`，任何依赖旧全局变量的代码将不再工作
- 树内拖拽功能不再提供（改为右键菜单"移动到…"）

## 🛡️ 开发流程强化

- 新增 4 条工作流规则：唯一性检查、日志优先、回调 Promise 化、WebView2 DnD 特殊性
- 复盘文档 `docs/postmortem-20250607.md`
