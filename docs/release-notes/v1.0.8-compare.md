# v1.0.8 — 开发者对比（技术细节）

> 此文件记录实现细节，供开发者参考，不展示给用户。

## 🔧 解析兼容性修复

| 问题 | 原因 | 修复 |
|------|------|------|
| YSM 加密模型解析失败 | `texture_width:512.0` 是 float64，struct 声明 int | 改为 `float64`，赋值时 `int()` |
| UV 格式不兼容 | YSM 输出 BlockBench 格式 UV 对象 `{...}` 非数组 `[...]` | Cube.UV 改为 `json.RawMessage` |
| 空骨骼崩溃 | Go nil slice 序列化为 `null`，前端 `for...of` 报错 | `make([]Cube2D, 0)` 保证 `[]` |
| `bone.cubes is not iterable` | 前端迭代缺 null 检查 | 所有 `bone.cubes` 加 `\|\| []` |
| `.zip` 内 YSM 模型 | zip 无 `minecraft:geometry` 时未尝试 YSMParser | zip/7z 失败后 fallback 到 YSMParser |

## 🧪 技术说明

- `.ysm` 是 YSM 专有二进制格式（YSGP header + AES 加密），非标准 zip
- 旧版 YSM 模组的 `UvUnion` 用 `@JsonAdapter` 同时支持数组/对象两种 UV 格式
- YSMParser 输出文件结构：`output/{name}/models/{main,arm}.json` + `textures/` + `animations/`
