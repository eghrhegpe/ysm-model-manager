## 🦴 多文件模型骨骼合并（实现细节）

三路合并路径统一：WASM / CLI / ZIP-7z 全部使用"全追加 → Build 内去重"策略。

## 🔧 2026-06-09 追加修复

| 问题 | 根因 | 修复 |
|------|------|------|
| ZIP 手臂位置偏移 | `pivots` map 只保留首次出现 pivot，但 arm.json(先) pivot 与 main.json(后) 不同 | `pivots` 优先保留**有 parent 的骨骼** pivot |
| 骨骼旋转丢失 | 去重时只更新了 parent/position，未更新 rotation | 去重同步更新 `LocalRotation` |
| cube 位置偏移 | bone 和 cube 用了不同 pivot | bone 和 cube 统一用 `pivots[b.Name]` |
| faceUV 被丢弃 | 前端 `parseBedrockGeometryFromJSON` 只处理 Array UV | 同时支持 Array UV / 对象 faceUV / 字符串 faceUV |
| 纹理尺寸错误 | 多几何体合并只取第一个文件的纹理尺寸 | 合并时保留**最大** texWidth/texHeight |
| 多纹理只用一个 | 所有 mesh 共用 `cubeTex`，未按骨骼选择纹理 | 按 `md.texIdx` 选对应纹理；多纹理模型强制走 JS 兜底 |
| 纹理文件名匹配失败 | `main.json` 找不到匹配纹理名 | 改为按**模型文件顺序**对应纹理顺序 |
