# v1.3.0 — 3D 渲染引擎翻修 + 多文件模型管线统一

> 🚧 草稿 — 尚未发布

## 🎯 核心改动

### 🦴 多文件模型骨骼合并重写

修复多 geometry 文件模型（main.json + arm.json）的骨骼合并逻辑，同步 YSMViewer 的 `ThreeJsPayloadBuilder.cs` 渲染管线：

| 问题              | 根因                                                 | 修复                                                |
| ----------------- | ---------------------------------------------------- | --------------------------------------------------- |
| 同名骨骼层级错误  | RightArm 在两个文件中都有，后加载的覆盖了正确 parent | 同名骨骼**保留首次层级**，仅追加 cubes              |
| 手臂立方体偏移    | `pivots` 共享 map 被同名骨骼的第二次出现覆盖         | cube 计算使用**当前骨骼自身 pivot**，不依赖共享 map |
| 骨骼旋转丢失      | `Bone2D` 缺少 `Rotation` 字段，JSON 解析未读取       | 新增 `Rotation [3]float64` + 解析 + 转四元数        |
| ZIP/7z 立方体丢失 | 合并时用 `seen` map 跳过同名骨骼，cubes 被静默丢弃   | 统一改为全追加，去重委托给 `threejs.Build()`        |

**三路合并路径统一**：WASM / CLI / ZIP-7z 全部使用"全追加 → Build 内去重"策略。

### 🎨 透明度渲染修复

- 透明纹理的透明区域不再遮挡后面模型
- `transparent: true` + `depthWrite: false` → `alphaTest: 0.5`
- Three.js 官方推荐方案：像素风贴图为二元透明度，alphaTest 丢弃透明像素，不参与深度缓冲
