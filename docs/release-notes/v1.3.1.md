# v1.3.1 — CSS 工程化 + Shadow DOM 样式隔离修复

> 发布日期: 2026-06-09

## 🎯 核心改动

### 🎨 CSS 搬迁移至 Shadow DOM（修复创意工坊/诊断页/设置页样式丢失）

之前将 CSS 从 `components.css`（全局）迁移到 `content-css.js`（Shadow DOM）时，大量 workshop 相关的 CSS 类被遗漏，导致全局 CSS **无法穿透 Shadow DOM**，创意工坊、诊断页、设置页出现不同程度的样式缺失。

| 页面                 | 缺失的样式                                                        | 表现                                            |
| -------------------- | ----------------------------------------------------------------- | ----------------------------------------------- |
| **创意工坊仓库**     | `.ws-header`、`.ws-btn*`、`.ws-search`、`.link-badge*` 等 20+ 类  | 按钮无边框/颜色、搜索框无样式、数据源徽标不显示 |
| **创意工坊模型列表** | `.ws-row`、`.ws-name`、`.ws-badge`、`.ws-dl-model` 等             | 行无边框/间距、按钮不可见                       |
| **创意工坊站点视图** | `.ws-scroll`、`.ws-preset-btn`、`.ws-action-btn`、`.ws-cr-row` 等 | 创作者编辑行/预设按钮/空状态无样式              |
| **创意工坊下载队列** | `.ws-queue-*`、`.ws-progress-*`                                   | 进度条/错误提示无样式                           |
| **诊断页空状态**     | `.stat-row`、`.diag-stat`、`.diag-stat-muted`                     | "暂无日志" 文字无字号/颜色/间距                 |
| **设置页链接提示**   | 硬编码 `#6c7086` 颜色 + `display=""` 无法覆盖 CSS                 | 链接模式提示文字（推荐/不推荐）完全不显示       |

### 🐞 Bug 修复

| 问题                          | 根因                                                | 修复                               |
| ----------------------------- | --------------------------------------------------- | ---------------------------------- |
| 模型行 `.ws-right` 列布局冲突 | 两处用了同名 `.ws-right`（列布局 vs 行右分组）      | 行右分组改名为 `.ws-row-right`     |
| 设置页链接提示从不显示        | `el.style.display = ""` 无法覆盖类的 `display:none` | 改为 `display = "block"`           |
| 链接提示文字颜色硬编码        | `color:#6c7086` 在浅色主题下不可见                  | 改为 `color:var(--muted,#6c7086)`  |
| 两个 `.hdr-btn` 规则重复      | content-css.js 中有两段完全相同的 `.hdr-btn` 定义   | 保留一份（不影响功能，仅代码冗余） |

### 📁 影响文件

| 文件                                                      | 改动                                                     |
| --------------------------------------------------------- | -------------------------------------------------------- |
| `frontend/js/components/app-content/content-css.js`       | **新增 ~80 条 CSS 规则** — 完整的 workshop/诊断/设置样式 |
| `frontend/js/components/app-content/workshop-render.js`   | 重命名 `.ws-right` → `.ws-row-right`                     |
| `frontend/js/components/app-content/workshop-settings.js` | 修复 `display=""` → `display="block"`                    |
| `frontend/css/components.css`                             | 删除已迁移的 workshop CSS（保留 `.ws-preview` 全局类）   |

## 📝 说明

本次发布不包含 3D 模型预览的改进（v1.3.0 草稿中的 3D 渲染引擎翻修将在后续版本正式发布）。

### 🔧 技术债清理

- **Shadow DOM 样式隔离规范化**：所有组件样式都放在对应 Shadow DOM 的 CSS 模块中，不再依赖全局 CSS
- **CSS 变量化**：硬编码颜色值逐步替换为 CSS 变量（`var(--txt)`、`var(--muted)` 等）
