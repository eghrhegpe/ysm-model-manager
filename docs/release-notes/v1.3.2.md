# v1.3.2 — 配置文件持久化修复 + 路径设置入口合并

> 发布日期: 2026-06-10

## 🎯 核心改动

### 🔧 `SaveAppConfig` 先写文件再校验

之前 `SaveAppConfig` 先对 mcRoot 执行 `ValidateMinecraftDir` 校验，校验失败就返回错误，**配置文件不会被创建**。这导致：

- 侧栏/设置页设置 mcRoot 后配置文件可能不生成
- 只有设 repoRoot（跳过校验）才能创建文件
- 如果 `ysm_config.json` 被删除，重启后无法自动恢复

**修复**：先 `os.WriteFile` 写入配置文件，再校验。校验失败只是不启动 watcher，不影响文件存在。

### 🚪 路径设置入口合并（去重）

侧栏和仓库树的快捷路径按钮改为**跳转到设置页**，不再自己弹目录选择器：

| 按钮                      | 之前              | 之后         |
| ------------------------- | ----------------- | ------------ |
| 侧栏「🎮 未设置」         | 弹目录选 mcRoot   | → 跳转设置页 |
| 仓库树「📁 未设置」       | 弹目录选 repoRoot | → 跳转设置页 |
| 设置页「🎮 游戏根目录」   | 浏览+自动检测     | 唯一入口     |
| 设置页「📁 模型仓库路径」 | 浏览              | 唯一入口     |

这样还修复了仓库树按钮丢失 mcRoot/linkMode 当前值的问题（之前写死空和 copy）。

### 🚀 启动时自动检测 mcRoot

如果 `ysm_config.json` 被删除或 mcRoot 为空，启动时自动扫描常见路径（`C:\PCL2\.minecraft`、`%APPDATA%\.minecraft` 等），找到后自动写入配置文件。

### 🐞 CSS 修复（延续 v1.3.1）

- **设置页链接提示从不显示** — 修复 `display` 覆盖问题
- **硬编码颜色** — 改为使用 CSS 变量
- **诊断页空状态无样式** — 添加对应 CSS 类
