# v1.3.3 — 纹理精度修复 + 旧格式兼容（无 ysm.json ZIP / YSGP V2）

> 发布日期: 2026-06-10

## 🎯 核心改动

### 🖼️ 纹理精度修复（PNG 实际尺寸覆盖几何体元数据）

几何体 JSON 的 `texture_width/height` 可能声明过小（如 32×32），但实际 PNG 为 2048×2048。旧代码只在 `texWidth === 0` 时用 PNG 尺寸覆盖，32 ≠ 0 所以跳过，导致 UV 归一化错误——纹理显示为极小一块。

**修复**：读取 PNG IHDR 获取实际尺寸，当 PNG 实际尺寸 > 几何体声明的尺寸时覆盖，不再仅当 `texWidth === 0` 时覆盖。

### 📦 无 `ysm.json` 的 ZIP 支持

部分 ZIP 模型包不含 `ysm.json`，只有 `main.json` + `arm.json` + 纹理（如 [就叫纸板] 円香 2023-04）。`ExtractYsmSummary()` 找不到 `ysm.json` 直接报错，导致前端无法解析。

**修复**：`ExtractYsmSummary` 新增降级扫描分支，自动检测 ZIP 内的模型文件、动画和纹理，用文件名作为展示名。

### 🔐 YSGP V2 早期模型兼容（BOM+文本头部变体）

2023 年早期模型使用 YSGP V2 格式变体：文件以 **UTF-8 BOM** + `"YSGP"` + 文本头部开头，之后才是加密数据。WASM 解析器无法识别此变体（如芙兰朵露、古明地觉等模型）。

**修复**：前端新增 `buildStdYsgpFromTextVariant()` 从文本头部提取密钥并重组为标准格式；Go 端 `detectYSGPHeader()` 和 `isYSGP()` 支持跳过 BOM 检测，三处联动修复。

### 🐞 其他修复

- `AnalyzeYSMHeader` 对 YSGP V2 返回空 — 新增 `detectYSGPHeader` 先检测魔数
- `isYSGP` 对带 BOM 的 YSGP 文件返回 false — 跳过 BOM 再检查

- Go: 1.22+ (Wails v2.12.0)
- 3D: Three.js r160
