## 起点：14 个"逆天设计"诊断（v1.5.3）

按前文分析的逆天设计，按**实际影响用户的程度**排列：

### 🔴 紧急（影响功能正确性）

| 优先级 | 问题                                                                            | 文件                         | 影响                                                                                                                   |
| ------ | ------------------------------------------------------------------------------- | ---------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| P0     | index.html 中 `window.bus` 不存在 → 首次启动导航默认页不跳转                    | index.html + app-modules.js  | 用户打开应用后可能卡在白页或错误页——因为没有 `nav:change` 事件发出。**已修复 ✅**                                      |
| P1     | handler-dnd.js 的 `setTimeout(50ms)` 跨组件时序依赖                             | handler-dnd.js               | 50ms 内如果 Shadow DOM 还没渲染完，`repo:switch-tab` 会发到无人监听的组件上，导入队列 tab 不切换。**已修复 ✅**        |
| P2     | `_loadPreviewImage` 和 `loadModelData` 双缓存竞争 → 解压 YSM 模型预览有时空骨骼 | index.js + preview-loader.js | WASM 返回空 geometry 后缓存被覆盖，Go AnalyzeBedrockModel 的正确骨骼被忽略，用户看到"⚠️ 未找到几何数据"。**已修复 ✅** |

### 🟡 中紧急（用户体验受损）

| 优先级 | 问题                                                                                        | 文件                                      | 影响                                                                                                 |
| ------ | ------------------------------------------------------------------------------------------- | ----------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| P3     | Go `importModelFile` 系列 6 个函数 → 新增扩展名时需改 6 处                                  | app_install.go                            | 加一个新扩展名要改 3 个 switch-case，上次加 `.json` 就漏了 `ImportModelFileOverwrite`。**已修复 ✅** |
| P4     | resource_types.json 与实际代码的扩展名列表不一致                                            | app_scan.go、sync.go、installer.go 等多处 | 改 JSON 不等于改功能——每处硬编码的扩展名列表都可能和 JSON 不同步                                     |
| P5     | 事件命名风格不统一：`nav:change` / `nav:changed` / `stats:refresh` / `import:pending-files` | 全局                                      | 新 AI 或开发者接手时需要猜"是 change 还是 changed"                                                   |

### 🟢 低紧急（代码可维护性）

| ------ | ----------------------------------------------------------------------------- | -------------------- | -------------------------------------------------------------------------------------- |
