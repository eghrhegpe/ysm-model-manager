## ✨ 新功能

### PrismLauncher 实例布局支持

来自社区贡献者 [@zuogeren1](https://github.com/zuogeren1) 的 PrismLauncher 支持，现在可以识别 PrismLauncher 特有的实例目录结构：

| 布局                 | 路径模式                               | 说明                          |
| -------------------- | -------------------------------------- | ----------------------------- |
| PrismLauncher 根目录 | `{prism}/instances/{name}/.minecraft/` | 直接选 PrismLauncher 安装位置 |
| instances 目录自身   | `{instances}/{name}/.minecraft/`       | 用户直接选了 instances 文件夹 |
| minecraft（无点）    | `{instances}/{name}/minecraft/`        | 兼容部分无点版本的 Prism 实例 |
| 标准 vanilla         | `.minecraft/versions/{name}/`          | 原版/PCL2/HMCL/BakaXL 不变    |

## ⚡ 性能优化

| 优化项                     | 文件/组件                             | 效果                                                                     |
| -------------------------- | ------------------------------------- | ------------------------------------------------------------------------ |
| `HasYSMMod` 文件名预过滤   | `go/ysm/ysm.go`                       | 先检查 JAR 文件名含 `yes_steve_model`/`ysm-` 才打开 ZIP，避免逐 JAR 解析 |
| `HasModInDir` 文件名预过滤 | `go/ysm/ysm.go`                       | 非 ysm 类型仅文件名匹配即可，跳过 ZIP 打开                               |
| 扫描缓存 TTL 2s → 30s      | `app_scan.go`                         | 减少重复磁盘扫描                                                         |
| sidebar 并发锁             | `app-sidebar/index.js`                | `_loading` 锁防止并发 reload                                             |
| stats:refresh 300ms 防抖   | `app-sidebar/index.js`                | 快速多次刷新时合并为一次                                                 |
| missing/extra 延迟构建     | `app-sidebar/loader.js` + `render.js` | 展开卡片时按需映射对象，大数据量不卡 UI                                  |
| statusList 提取到循环外    | `handler-sync.js`                     | N+1 次 API 调用 → 1 次                                                   |
