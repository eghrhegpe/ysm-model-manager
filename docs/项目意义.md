# YSM 模型管理器 — 项目意义

## 起源

这个项目始于一篇分析 YSM 生态的长文（43 小项、400+ 评论归类）。文章剖析了加密模型争议中各方的立场与利益，最终结论是：**玩家需要替代方案，需要掌握对自己模型的控制权。**

市面上已有的工具（YSMViewer、网页浏览器）停留在"查看模型"。这个项目把"看 + 管 + 同步"做全了——这不是技术上的巧合，是那篇文章的逻辑推导出来的必然结果。

## 核心意义

### 1. 让"用脚投票"成为可能

文章说抵制高价加密模型，但玩家没有替代工具时只能忍着用。这个项目支持仓库管理、整合包同步，让玩家可以自己搭建模型库，摆脱对特定作者/平台的依赖。

### 2. 开源的管理工具本身就是立场

项目采用 Wails + 开源，把解析、管理、同步全链路的控制权交给用户。加密模型的辩护者常说"不开源是为了保护创作者"，这里用代码证明了：**开源和管理可以共存。**

### 3. 跨出"浏览"的范畴

从查看模型，进化为**管理整个模型资产生命周期的平台**：

| 环节 | 功能 |
|------|------|
| 导入 | 支持 .ysm / .zip / .7z / 文件夹 |
| 浏览 | 2D 骨骼图 + 3D 多纹理预览 |
| 管理 | 仓库树、筛选、回收站 |
| 部署 | 整合包同步、一键安装 |
| 发布 | 创意工坊站点浏览、下载 |

### 4. 在生态中占据"基础设施"位置

YSM 圈子虽小，但这个项目覆盖了导入→浏览→管理→同步的完整链路。天花板取决于 YSM 生态本身的大小，但项目已经在生态中占据了基础设施的生态位——这是最值钱的生态位。

## 项目立场

站在玩家一边，提供实用的、开源的工具，而不是参与社区争论。

> "用脚投票：不买、不传播、不美化吃相难看的模型。支持替代方案，理性发声。"
